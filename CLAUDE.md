# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

This is a [Quarto](https://quarto.org/) website for **GENE 46100: Deep Learning in Genomics**, a course taught at UChicago by Haky Im and Ran Blekhman. The site is rendered to the `docs/` directory and deployed via GitHub Pages.

## Key Commands

### Build / Render

```bash
# Render the entire site
quarto render

# Preview the site locally with live reload
quarto preview

# Render a single file
quarto render post/unit00/homework-01.qmd

# Render slides specifically (excluded from default render, must be done explicitly)
quarto render post/unit00/slides-unit00.qmd --to revealjs
```

> **Note:** The default render in `_quarto.yml` only includes `post/unit00/**/*.qmd`. Units 01–03 are **not rendered by default** — to include them, add their glob patterns to the `render:` list in `_quarto.yml`. Files matching `**/slides*.qmd` are also excluded; render slides individually with `--to revealjs`.

### Environment Setup

The course uses a conda environment named `gene46100` (Python 3.12). Packages are installed via `%pip` inside notebooks, not via conda.

```bash
conda create -n gene46100 python=3.12
conda activate gene46100
conda install nb_conda_kernels jupyter_server ipykernel
pip install nbclient nbformat
```

For `.qmd` files with Python code blocks, add this to the YAML frontmatter:

```yaml
jupyter:
  kernelspec:
    name: "conda-env-gene46100-py"
    language: "python"
    display_name: "gene46100"
```

## Architecture

### Site Structure

- `_quarto.yml` — site config: output to `docs/`, theme cosmo, katex math
- `index.qmd` — homepage listing (shows non-draft posts only, sorted by date)
- `post/_metadata.yml` — shared defaults for all posts: `freeze: auto`, code-fold off, code-tools on
- `docs/` — rendered HTML output (committed to git, served via GitHub Pages)
- `post/` — source content organized by unit

### Content Organization

Course content lives in `post/` and is organized by unit:

| Directory | Topic |
|-----------|-------|
| `post/unit00/` | Intro to DL, DNA basics, TF binding prediction |
| `post/unit01/` | Language models, GPT (nanoGPT), attention |
| `post/unit02/` | Sequence-to-function models (Enformer, Borzoi) |
| `post/unit03/` | Single-cell RNA (scGPT, scanpy, Seurat) TO BE REPLACED WITH MICROBIOME CONTENT|

Each unit folder typically contains:
- `index.qmd` — unit overview/setup page (often `draft: true`)
- Homework `.qmd` files
- Hands-on notebooks (`.ipynb` or `.qmd`)
- A `slides-*.qmd` for RevealJS slide decks

### Draft System

Posts with `draft: true` in their YAML frontmatter are hidden from the main `index.qmd` listing but appear in `index-all.qmd`. Use `draft: true` for in-progress content.

### Freeze System

`post/_metadata.yml` sets `freeze: auto` — Quarto caches executed notebook outputs in `.quarto/_freeze/` so code doesn't re-run on every render unless the source changes. To force re-execution, delete the relevant `.quarto/_freeze/` cache entry or set `freeze: false` in the document.

### Slides

RevealJS slide decks use `format: revealjs` with `chalkboard: true`. They are intentionally excluded from the bulk render (`!**/slides*.qmd` in `_quarto.yml`) and must be rendered individually.

Stub pages (`page-slides-*.qmd`) are auto-generated by `scripts/generate-slide-stubs.py` and should not be edited directly — edit the generator script instead. Run via `bash render-all.sh`.

> **Warning:** Never leave stale `.html` files in `post/` (e.g. from old renamed slide decks). Quarto copies all `.html` files it finds in source directories into `docs/`, polluting the output. `render-all.sh` includes a cleanup step for this, but if you manually render slides, delete the generated HTML from `post/` afterward.

#### Slide CSS

`custom.css` lives at the **project root** (`/custom.css`). All slide files reference it as `css: ../../custom.css` (slides are two levels deep at `post/unitXX/`). Do not create per-unit copies.

The canonical source for this CSS across all projects is `~/.claude/templates/quarto-slides/custom.css`.

Available CSS classes:

| Class | Purpose |
|-------|---------|
| `.source-credit` | Attribution pill, bottom-right. Place div immediately after `##` heading, before any `. . .` markers. |
| `.source-credit-linked` | Link-only attribution, no background pill. |
| `.takehome` | Blue highlighted box floating above the footer. |
| `.columns.top` | Two-column layout, top-aligned (default is vertically centered). |

#### Slide Content Rules

- Slides cover **prerequisite concepts only** — not results that students will discover in the notebook
- Use `> **Discuss:**` blockquotes for instructor prompts (in **notebooks**, not slides)
- Exercises go in `homework-NN.qmd`, not slide decks
- Use `# Part N: Title {background-color="#1e3a5f" style="color:white;"}` for section dividers

To export notebooks as `.ipynb` to Google Drive: `bash scripts/export-notebooks.sh [unit]` (defaults to `unit00`).

## Content Conventions

### File naming

Each unit uses these filename prefixes, numbered within the unit:

| Prefix | Type |
|--------|------|
| `slides-NN-` | RevealJS lecture slides |
| `page-slides-NN-` | Auto-generated stub page linking to slide deck |
| `notebook-NN-` | Jupyter/Quarto notebook |
| `homework-NN` | Homework assignment |
| `challenge-NN-` | Challenge problem |
| `setup-` / `guide-` | Reference/resource pages (no number prefix) |

### Title numbering

Slides and notebooks are prefixed with `NN ·` (e.g., `"01 · Quick Introduction to Deep Learning"`) so the listing page guides students through the intended order. Guide, challenge, and homework files are **not** numbered.

Notebook titles should **match their paired slides title exactly** (same number, same wording). The SLIDES/NOTEBOOK badges in the listing already distinguish them.

### Section dividers in slides

Use dark-background section divider slides to chunk long decks into named parts:

```markdown
# Part N: Title {background-color="#1e3a5f" style="color:white;"}
```

Current section structure:
- **slides-01**: Parts 1–6 (Linear Models, Gradient Descent, The ML Framework, Neural Networks, From Playground to PyTorch, Wrap-Up)
- **slides-02**: Parts 1–4 (How CNNs Work, Putting It Together, Using CNNs in Practice, Summary)
- **slides-03**: Parts 1–3 (The Task, DNA → Numbers, Linear vs CNN)

### Exercises belong in notebooks, not slides

Do not add exercises to slide decks. Instead:
- Use `> **Discuss:**` blockquotes in notebooks for instructor prompts / discussion questions
- Use `homework-NN.qmd` for graded, open-ended problems

Slides should focus on concept delivery, figures, and key ideas only.
