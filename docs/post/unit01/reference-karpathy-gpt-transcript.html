Narrative Arc

A single ~2-hour live-coding session that builds a character-level GPT from an empty file. The arc moves from the simplest possible model outward, adding one component at a time until the full Transformer is assembled and trained.

Timestamp Section What is added
0:00–7:00 Motivation ChatGPT → “Attention is All You Need” (2017) → nanoGPT repo overview; sets up Tiny Shakespeare as the toy dataset
7:00–22:00 Data & tokenization Load 1M-char Shakespeare; character vocabulary (65 tokens); stoi/itos encode/decode; train/val split; block_size + batch_size batching; the “32 independent examples in one batch” insight
22:00–38:00 Bigram baseline Simplest nn.Module; embedding table as logit lookup; cross-entropy loss; generation loop; AdamW training loop; estimate_loss on train/val splits
38:00–49:00 Port to script Refactor notebook → .py; GPU support; cleaner hyperparameter block
49:00–57:00 The attention trick (3 versions) Version 1: for-loop averaging; Version 2: lower-triangular matrix multiply (same result, no loop); Version 3: softmax + -inf masking — the exact form used in real attention
57:00–~80:00 Self-attention (single head) Q/K/V projections; scaled dot-product q @ k.T * head_size**-0.5; causal mask; softmax → weighted sum of values; “keys and queries don’t talk across batches”
80:00–95:00 Multi-head attention Run n_head heads in parallel; concatenate outputs; linear projection back to n_embd
95:00–110:00 Feed-forward block + residuals + LayerNorm 2-layer MLP per position; x = x + sublayer(ln(x)) residual form; pre-norm (LayerNorm before sublayer, differs from original paper)
110:00–120:00 Positional embeddings Learned position table added to token embeddings; why attention alone is permutation-invariant
~120:00–end Scaling up Dropout; full hyperparameter table (small → GPT-2-scale); loss curve comparison; relationship to GPT-2’s 124M-parameter model

Key throughline: each addition is motivated by a concrete deficiency in the previous model. The bigram model can’t see context → add averaging → averaging ignores relevance → add data-dependent weights → that’s attention. The lesson never introduces a component without first showing why the simpler version fails.

Not covered: fine-tuning, RLHF, subword tokenization (BPE), or any genomics application — those are extensions in the course notebooks.

## Narrative Arc

A single ~2-hour live-coding session that builds a character-level GPT from an empty file. The arc moves from the simplest possible model outward, adding one component at a time until the full Transformer is assembled and trained.

| Timestamp | Section | What is added |
|-----------|---------|---------------|
| 0:00–7:00 | **Motivation** | ChatGPT → "Attention is All You Need" (2017) → nanoGPT repo overview; sets up Tiny Shakespeare as the toy dataset |
| 7:00–22:00 | **Data & tokenization** | Load 1M-char Shakespeare; character vocabulary (65 tokens); `stoi`/`itos` encode/decode; train/val split; `block_size` + `batch_size` batching; the "32 independent examples in one batch" insight |
| 22:00–38:00 | **Bigram baseline** | Simplest `nn.Module`; embedding table as logit lookup; cross-entropy loss; generation loop; AdamW training loop; `estimate_loss` on train/val splits |
| 38:00–49:00 | **Port to script** | Refactor notebook → `.py`; GPU support; cleaner hyperparameter block |
| 49:00–57:00 | **The attention trick (3 versions)** | Version 1: for-loop averaging; Version 2: lower-triangular matrix multiply (same result, no loop); Version 3: softmax + `-inf` masking — the exact form used in real attention |
| 57:00–~80:00 | **Self-attention (single head)** | Q/K/V projections; scaled dot-product `q @ k.T * head_size**-0.5`; causal mask; softmax → weighted sum of values; "keys and queries don't talk across batches" |
| ~80:00–~95:00 | **Multi-head attention** | Run `n_head` heads in parallel; concatenate outputs; linear projection back to `n_embd` |
| ~95:00–~110:00 | **Feed-forward block + residuals + LayerNorm** | 2-layer MLP per position; `x = x + sublayer(ln(x))` residual form; pre-norm (LayerNorm before sublayer, differs from original paper) |
| ~110:00–~120:00 | **Positional embeddings** | Learned position table added to token embeddings; why attention alone is permutation-invariant |
| ~120:00–end | **Scaling up** | Dropout; full hyperparameter table (small → GPT-2-scale); loss curve comparison; relationship to GPT-2's 124M-parameter model |

**Key throughline:** each addition is motivated by a concrete deficiency in the previous model. The bigram model can't see context → add averaging → averaging ignores relevance → add data-dependent weights → that's attention. The lesson never introduces a component without first showing why the simpler version fails.

**Not covered:** fine-tuning, RLHF, subword tokenization (BPE), or any genomics application — those are extensions in the course notebooks.
